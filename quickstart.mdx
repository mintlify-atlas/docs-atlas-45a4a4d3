---
title: "Quickstart: Run Levi.dev portfolio locally"
description: "Clone the repository, configure your environment variables, run database migrations, and start the development server in under ten minutes."
---

This guide walks you through getting the application running on your local machine from scratch. You will need Node.js 18+, a MySQL database (local or hosted), and accounts for Stripe and PayPal to complete payment configuration.

<Steps>
  <Step title="Clone the repository">
    Clone the repository and navigate into the project directory:

    ```bash
    git clone https://github.com/ImLevii/portfolio-v0.git
    cd portfolio-v0
    ```
  </Step>

  <Step title="Install dependencies">
    Install all dependencies with your preferred package manager. The `postinstall` script automatically runs `prisma generate` after installation.

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Copy or create a `.env.local` file in the project root. The following variables are required — the build will fail without them:

    ```bash .env.local
    # Database
    DATABASE_URL="mysql://user:password@localhost:3306/portfolio"

    # Authentication (NextAuth.js)
    AUTH_SECRET="your-random-secret-min-32-chars"

    # GitHub OAuth (optional, enables GitHub sign-in)
    AUTH_GITHUB_ID="your-github-oauth-app-client-id"
    AUTH_GITHUB_SECRET="your-github-oauth-app-client-secret"

    # Google OAuth (optional, enables Google sign-in)
    AUTH_GOOGLE_ID="your-google-oauth-client-id"
    AUTH_GOOGLE_SECRET="your-google-oauth-client-secret"

    # Stripe
    NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY="pk_test_..."
    STRIPE_SECRET_KEY="sk_test_..."
    STRIPE_WEBHOOK_SECRET="whsec_..."

    # PayPal
    NEXT_PUBLIC_PAYPAL_CLIENT_ID="your-paypal-client-id"
    NEXT_PUBLIC_PAYPAL_CLIENT_SECRET="your-paypal-client-secret"

    # App URL (required in production to prevent redirect issues)
    NEXT_PUBLIC_APP_URL="http://localhost:3000"
    ```

    <Warning>
      `AUTH_SECRET`, `DATABASE_URL`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, `STRIPE_SECRET_KEY`, `NEXT_PUBLIC_PAYPAL_CLIENT_ID`, and `NEXT_PUBLIC_PAYPAL_CLIENT_SECRET` are all required. The pre-build check script (`scripts/check-env.js`) will exit with an error if any of these are missing.
    </Warning>

    <Tip>
      Generate a strong `AUTH_SECRET` with: `openssl rand -base64 32`
    </Tip>
  </Step>

  <Step title="Run database migrations">
    Apply the Prisma schema to your database. This creates all tables defined in `prisma/schema.prisma`:

    ```bash
    npx prisma migrate dev --name init
    ```

    If you are connecting to an existing database and want to sync without creating a migration file, use:

    ```bash
    npx prisma db push
    ```

    <Note>
      The database provider is MySQL. Ensure your `DATABASE_URL` points to a running MySQL instance before running migrations.
    </Note>
  </Step>

  <Step title="Seed initial data (optional)">
    A seed script at `prisma/seed.js` inserts three sample products (Developer Starter Kit, Pro Icon Pack, Ultimate UI Bundle). Run it with:

    ```bash
    node prisma/seed.js
    ```

    The script uses `upsert` so it is safe to run multiple times. You can also open Prisma Studio to inspect or add data directly:

    ```bash
    npx prisma studio
    ```

    Prisma Studio opens at `http://localhost:5555` and lets you insert rows into any table without writing SQL.

    <Tip>
      To make yourself an admin, sign in once to create your user record, then run `node prisma/promote-admin.js` (after editing the email constant in the file) or find your row in Prisma Studio and change the `role` field from `CUSTOMER` to `ADMIN`.
    </Tip>
  </Step>

  <Step title="Start the development server">
    Start the Next.js development server:

    ```bash
    npm run dev
    ```

    The app is now running at `http://localhost:3000`.

    | Route | Description |
    |---|---|
    | `/` | Public portfolio homepage |
    | `/shop` | Digital product storefront |
    | `/auth/signin` | Sign-in page |
    | `/admin` | Admin dashboard (ADMIN role required) |
    | `/settings` | User account settings |
    | `/licenses` | User license key management |
  </Step>
</Steps>
